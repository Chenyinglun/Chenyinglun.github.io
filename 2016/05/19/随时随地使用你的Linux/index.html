    打造一台专属于你的服务器

    作为一个爱编程的自动化工程师，有时候在外地闲时想摆弄一下Linux，完善自己的项目怎么办呢？出到外面不一定带着电脑，虽然说笔记本够轻了的。在别人电脑不习惯以及各种文件又要同步。

    本文能够帮助你在远处ssh登录你的Linux

    • 学习编程和linux
    • 维护本地项目
    • 添加系统任务（例如下载一部电影）
    • 调用linux系统的强大工具
    • 作GPIO服务器的基础（树莓派）
    • 安全监控
    • 以及其他各种等等

    这是最终登上去的结果和普通局域网ssh是一样的。如图，我们使用Linux不就是要用到它的文字终端么。

    在网络和数字化的现代，可以说，以树莓派搭建的家庭服务器，基本就是智能家居的雏形。

    准备的东西

    1. 树莓派（或任何运行Linux的电脑）
    2. 一个联网的路由器

    选择树莓派的原因是它3.5w功耗，大概两周才一度电，十分低，而且还具备桌面服务器没有的gpio口。
    其他外设按需要添加。

    例如在电路方面,gpio 0 连接继电器控制电水壶，在回家之前发送gpio 0 write 1便接通，回家即可泡茶~

    配置过程

    系统配置

    首先在树莓派上的配置（debian、ubuntu同理）

    在raspbian里官方已经自带ssh server，其他的请自行安装例如：

    $ sudo apt-get install openssh-server

    在本地局域网里请自行测试是否可行，如失败，请检查ssh设置及防火墙设置。

    设置静态ip
    提供一种最直接在设置静态ip的方法
    用编辑器打开 /etc/network/interfaces

    $ sudo nano /etc/network/interfaces

    修改例如：

    iface wlan0 inet static  #这里用的是树莓派自带的wifi
#yournetwork
address 192.168.0.54   #你要设置的静态ip
netmask 255.255.255.0
gateway 192.168.0.1    #你的网关
network 192.168.0.1
 wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf

    这样就可以了，也可以同过路由器设置静态ip，树莓派通过dhcp获取。

    外网配置

    现在主流的路由器都自带DDNS和DMZ主机这个功能，这个我是在腾达上做的测试。其他的路由器请查看是否支持，如果没有的路由器建议刷openwrite.

    1. 申请一个免费的动态解析域名DDNS。
      这里用的是pubyun

    2. 在路由器里的DDNS选项里填入你注册的账号以及动态域名，然后确定。

    回到你你的域名管理页面，发现公网ip是随着你的路由器的变化而变化的。

    1. 在DMS主机上把树莓派刚才设置的静态ip写上，作用是把树莓派映射到外网。出于安全考虑，尽量保持你动态域名的隐匿。如不能，给主机请设置复杂密码以及安装防火墙。

    2. 检查是否ping通。

      ping 你的域名

    外出时要准备的东西

    多数场合手机即可。

    有代码量大的不妨可以带个小型蓝牙键盘。然后：

    1. 装上ssh软件，这里推荐juicessh或者connectbot

    其他联网非自己电脑情况。

    适合需要长时间工作。如果有U盘，把putty装上带走，仅484KB而已。忘记带就直接搜索下载，这个是开源软件，十分容易找到。

    登录方式

    如一般在局域网内ssh的方法。

    用户名@地址：22      #ssh默认端口是22

    写在最后

    有时候连不上的话，是因为动态域名没有更新及时。还没有经过长时间测验，如果需要工作长时间ssh编写文件的话，建议安装screen或者Tmux。

    这是本人第一次向外网连接的尝试及基础服务器的探索。目的是为了更好地了解Linux精髓和提高编程思维及能力。这仅仅是一个开始，不难发现，这便是智能家居的一种方案：以家庭服务器为中心搭建。这概念很早就有已有许多人所认同。难的是，家庭电器的自动化需要从硬件上精巧地设计以及以某种协议统一，虽然现在也有某种物联的协议。但许多是不开源的，相当于要买模块。今后有空再探索智能家庭的自动化最有效的方案。
