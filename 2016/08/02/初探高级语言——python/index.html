初探高级语言——python

2016.08.02 陈影伦 学习

题外话

有一个月没更新过了吧，课程什么的都才刚匆忙结束，毕竟准大三（笑）。难得偷闲写个博客，发觉还是比较多东西想写。先说下我在编程上的探索吧。

关于Python

接触了几个月的python，我想先说说我的对它的整体感觉：

  • 基于C语言，语法习惯比较容易迁移
  • 无时无刻要注意的缩进格式，即能优化代码的可读性，又能分块
  • 每个元素都是对象，十分适合面向对象编程
  • 开发效率高，运行效率低
  • 没有C上面的新手难题，入门快。

有了C，为什么还需要python

曾经一度天真的我认为，只要学会了C，就天下无敌了。其实真正是，精通C，熟练算法。仅仅是指针和链表就够新手原地好几个月了。摸爬滚打的我也是学会了C，掌握真正的算法要不断地历练。所以说学习曲线比较陡峭。
python的话，如果你有一定的编程基础，不到一个星期就可以入门了。
并不是因为简单，而是因为解释器帮我们做了在C语言上需要做的事情。有些工作是重复而繁琐的，python正好是简约。

既然是面向对象编程，所以，在一个项目里，python的作用很大就是在写上位机程序上。C语言的面向对象编程，一般就指C++了，这时python的优越性体现出来了，就是集成度高。就好比一个模拟电路，可以用集成芯片做出也可以用散零件拼接，集成芯片稳定，可靠性好，缩短开发周期，还节约地方。如今电脑、服务器的性能已经够强大，python即时运行效率不高，但是可以忽略不计。google服务器网站后台都用的python，能够质疑么？高级语言的发展，这也是如今软件发展得快的原因。

语言，够用就好

并不是所有的语言都需要去接触，去精通。这样只会竹篮打水一场空。我仅仅是希望，在低级语言，我有汇编（这算哪门语言？！），在中级语言，我有C语言，在高级语言，我有python。实际上是一样的，都是转换成01的机器码执行。但是有了这些，我才可能站在更高的维度里完成项目。
